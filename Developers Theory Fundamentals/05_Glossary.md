# 05. Glossary

Ниже собраны базовые определения, которые должны остаться после прохождения трека.

## 1. Клиент

Клиент — это программа или сервис, через который пользователь взаимодействует с системой и отправляет запросы.

## 2. Сервер

Сервер — это часть системы, которая принимает запросы, выполняет логику, работает с данными и возвращает ответы.

## 3. База данных

База данных — это хранилище данных приложения.

## 4. Клиент-серверная архитектура

Клиент-серверная архитектура — это модель, в которой клиент отправляет запрос, сервер его обрабатывает и возвращает ответ, при необходимости работая с базой данных.

## 5. HTTP

HTTP — это протокол, через который клиент и сервер обмениваются сообщениями в вебе.

## 6. HTTP-запрос

HTTP-запрос — это сообщение от клиента к серверу.

## 7. HTTP-ответ

HTTP-ответ — это сообщение от сервера к клиенту.

## 8. HTTP-метод

HTTP-метод — это указание, какое действие клиент хочет выполнить над ресурсом.

На базовом уровне важно понимать:
- `GET`
- `POST`
- `PATCH`
- `PUT`
- `DELETE`

## 9. Код состояния

Код состояния — это часть HTTP-ответа, которая показывает результат обработки запроса.

## 10. HTTPS

HTTPS — это HTTP с защищенной передачей данных.

## 11. HTML-документ

HTML-документ — это разметка страницы, которую получает и читает браузер.

## 12. DOM

DOM — это объектная модель документа, то есть внутренняя структура страницы в памяти браузера.

## 13. Узел DOM

Узел DOM — это отдельный объект внутри DOM-структуры, например элемент, текст или сам документ.

## 14. JavaScript и DOM

JavaScript может обращаться к DOM, находить элементы и изменять содержимое страницы.

## Минимум, который должен остаться после трека

Если участник прошел трек успешно, он должен уметь без терминологической путаницы:
- объяснить разницу между клиентом и сервером;
- объяснить роль базы данных в клиент-серверной архитектуре;
- объяснить, что такое HTTP-запрос и HTTP-ответ;
- назвать базовые HTTP-методы;
- объяснить, что такое код состояния;
- объяснить, что такое DOM;
- объяснить, как JavaScript меняет страницу через DOM.
