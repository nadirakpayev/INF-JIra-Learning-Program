# 02. SQL Basics

## Зачем нужен этот модуль

Этот модуль соответствует следующему шагу после фундаментальных основ: понять, что такое SQL-запрос и как он читается в базовом синтаксисе.

Во всех SQL-примерах ниже используется общий учебный датасет из [00_Mini_Dataset.md](00_Mini_Dataset.md).

## Что нужно понять

### 1. Что такое SQL-запрос

SQL-запрос — это инструкция к СУБД.

Через SQL-запрос можно:
- создавать структуру данных;
- вставлять данные;
- читать данные;
- изменять данные;
- удалять данные.

На практике для джуна особенно важны:
- `CREATE TABLE`
- `INSERT`
- `SELECT`
- `UPDATE`
- `DELETE`

Простой пример:
Если у нас есть база данных соцсети, то SQL-запрос похож на просьбу: "покажи все посты пользователя", "добавь новый комментарий" или "обнови описание профиля".

SQL-пример:
```sql
SELECT *
FROM posts;
```

Этот запрос означает:
"Покажи все записи из таблицы постов".

Подробнее:
- [SQL Academy, Вводная информация о SQL](https://sql-academy.org/ru/guide/intro-sql)

### 2. Базовый синтаксис SQL-запроса

На базовом уровне полезно понимать типовую логику:
- `SELECT` — что выбрать;
- `FROM` — откуда выбрать;
- `WHERE` — по какому условию;
- `ORDER BY` — как отсортировать.

Простой пример:
Фраза "покажи логины пользователей из Алматы и отсортируй по имени" почти дословно превращается в структуру SQL-запроса.

SQL-пример:
```sql
SELECT username
FROM users
WHERE city = 'Almaty'
ORDER BY username;
```

Подробнее:
- [SQL Academy, Базовый синтаксис SQL запроса](https://sql-academy.org/ru/guide/basic-syntax-sql-query)

### 3. SELECT

`SELECT` — базовая команда для чтения данных.

Нужно понимать:
- откуда берутся данные;
- какие столбцы выбираются;
- по каким условиям строки попадают в результат;
- как сортируется итог.

Простой пример:
`SELECT username FROM users` означает: "возьми из списка пользователей только колонку с логинами".

Подробнее:
- [PostgreSQL Querying a Table](https://www.postgresql.org/docs/14/tutorial-select.html)

SQL-пример:
```sql
SELECT username, created_at
FROM users;
```

Этот запрос означает:
"Покажи логин и дату создания аккаунта у всех пользователей".

### 4. FROM

`FROM` показывает источник данных.

Это может быть:
- одна таблица;
- несколько таблиц;
- подзапрос.

Простой пример:
Если вы говорите "покажи посты из списка постов", то таблица `posts` и есть источник данных. В SQL за это отвечает `FROM`.

SQL-пример:
```sql
SELECT title
FROM posts;
```

Здесь `posts` после `FROM` говорит, откуда именно брать данные.

### 5. WHERE

`WHERE` отбирает только нужные строки.

Без этого знания невозможно:
- фильтровать данные;
- искать записи по условию;
- проверять корректность тестового датасета.

Простой пример:
`WHERE likes_count > 100` означает: "покажи только те посты, у которых больше 100 лайков".

SQL-пример:
```sql
SELECT title, likes_count
FROM posts
WHERE likes_count > 100;
```

### 6. ORDER BY

`ORDER BY` задает порядок вывода.

Это особенно полезно:
- при проверке seed-данных;
- при чтении результатов запросов;
- при отладке модели.

Простой пример:
Если лента длинная, ее удобно отсортировать по дате публикации или по числу лайков. Именно это и делает `ORDER BY`.

SQL-пример:
```sql
SELECT title, created_at
FROM posts
ORDER BY created_at DESC;
```

Этот запрос означает:
"Покажи посты от новых к старым".

### 7. Aggregate functions

Агрегатные функции помогают считать:
- количество;
- сумму;
- среднее;
- минимумы и максимумы.

Они нужны для:
- быстрой проверки данных;
- элементарной аналитики;
- построения validation queries.

Простой пример:
`COUNT(*)` отвечает на вопрос: "сколько всего комментариев у этого поста?"

Подробнее:
- [PostgreSQL Aggregate Functions](https://www.postgresql.org/docs/current/functions-aggregate.html)

SQL-пример:
```sql
SELECT COUNT(*)
FROM comments
WHERE post_id = 15;
```

Этот запрос означает:
"Посчитай, сколько комментариев у поста с `id = 15`".

### 8. UPDATE и DELETE

Нужно понимать не только чтение, но и изменение данных.

Особенно важно:
- `UPDATE` меняет существующие строки;
- `DELETE` удаляет строки;
- ошибка в условии может затронуть слишком много данных.

Простой пример:
Если забыть условие в `DELETE`, это похоже на ситуацию, когда вы хотели удалить один спам-комментарий, а случайно стерли вообще все комментарии под постом.

SQL-пример:
```sql
UPDATE users
SET bio = 'Product designer and traveler'
WHERE id = 7;
```

```sql
DELETE FROM comments
WHERE id = 99;
```

В обоих случаях условие помогает изменить или удалить только нужную запись.

## Что нужно уметь после модуля

После этого модуля участник должен уметь:
- дать определение SQL-запроса;

- объяснить роль `FROM`, `WHERE` и `ORDER BY`;
- сделать простой `INSERT`;
- объяснить разницу между чтением и изменением данных.

- объяснить, почему `UPDATE` и `DELETE` без условия опасны;
- назвать разницу между `COUNT(*)` и обычным выводом строк.
